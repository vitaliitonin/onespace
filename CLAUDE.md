# Onespace — Product Brief

## Концепція

Onespace — україномовна платформа-агрегатор для пошуку фільмів та мультфільмів з поділом на дитячий та дорослий контент. Не стрімінг — виключно пошук, рейтинги та відкриття нового контенту. Після знаходження фільму користувач переходить на зовнішні платформи (Netflix, Kino UA тощо).

## Аудиторія

- Вік: 12–60 років
- Пріоритет: рейтинги та комфорт перегляду
- Географія: Україна
- Мова інтерфейсу: Українська

## Ключові функції

### Поділ контенту
- Два режими з перемикачем: **Дорослий** / **Дитячий**
- Вихід із дитячого режиму потребує PIN-коду або підтвердження
- Дитячий режим — платна функція (freemium)

### Пошук та відкриття
- Швидкий пошук за назвою
- Фільтри: жанр, рік, рейтинг IMDb, OTT-платформа
- Тренди та добірки (популярне, топ тижня, нові надходження)
- Рейтинги IMDb на кожній картці фільму

### Персоналізація (платно)
- Watchlist — особистий список "хочу подивитись"
- Дитячий профіль / PIN-режим
- Розширені фільтри (за жанром, роком, OTT-платформою)

## Бізнес-модель

| Tier | Функції |
|------|---------|
| **Free** | Анонімний пошук, рейтинги, тренди |
| **Premium** | Watchlist, дитячий PIN-режим, розширені фільтри |

- Монетизація через підписку (freemium)
- Можливий affiliate-дохід від посилань на OTT-платформи

## Джерела даних

- **TMDB API** — постери, описи, трейлери, жанри, дати виходу
- **IMDb datasets** — офіційні рейтинги та metadata

## Технічний стек (рекомендований)

| Шар | Технологія | Причина |
|-----|-----------|---------|
| Frontend | Next.js 14+ (App Router) | SSR для SEO, TypeScript |
| Стилі | Tailwind CSS | Темна тема, швидка розробка |
| Auth | NextAuth.js | Google/Email login з коробки |
| БД | PostgreSQL + Prisma | Watchlists, акаунти |
| Кеш | Redis | Кешування TMDB-відповідей |
| Деплой | Vercel | Безшовна інтеграція з Next.js |

## Дизайн-напрямок

- **Стиль**: темний, кінематографічний (IMDb / Letterboxd)
- **Теми**: Dark mode за замовчуванням, підтримка Light mode
- **Акценти**: постери фільмів як головний візуальний елемент
- **UX-пріоритет**: мінімум кліків до результату, чіткі рейтинги

## Платформи

1. **Desktop** — пріоритет на MVP
2. **Mobile** — наступний етап після Desktop

## Часові рамки

- **MVP**: 2–3 місяці
- MVP включає: пошук, рейтинги, тренди, режими контенту, базовий freemium

## Структура MVP (in scope)

- [ ] Головна сторінка з трендами
- [ ] Пошук з фільтрами (жанр, рік, рейтинг)
- [ ] Картка фільму (постер, опис, рейтинг IMDb, посилання на OTT)
- [ ] Перемикач Дорослий / Дитячий режим
- [ ] Реєстрація / вхід (email або Google)
- [ ] Watchlist (тільки для зареєстрованих)
- [ ] PIN-захист дитячого режиму (тільки Premium)

## Out of scope для MVP

- Власний стрімінг
- Рекомендаційний алгоритм на основі ML
- Соціальні функції (відгуки, коментарі)
- Мобільний застосунок
- Мультимовність (спочатку тільки UA)

## IA — Інформаційна архітектура

> Детально: `sitemap.md`, `flows.md`, `ia.html`

### Sitemap верхнього рівня (8 екранів)

| Рівень | Екран | Jobs |
|--------|-------|------|
| Глобальна | **Головна** | Main, J4, J1, E1 |
| Глобальна | **Пошук** | Main, J3, J1, J2 |
| Глобальна | **Дітям / Вийти** | J2, E1 |
| Контекстна | **Картка фільму** | Main, J1, J5, J2 |
| Контекстна | **Повний список добірки** | Main, J4, J1 |
| Контекстна | **PIN-екран** | J2, E1 |
| Глибока | **Налаштування PIN** | J2, E1 |
| Глибока | **Реєстрація / Вхід** | J2 |
| Глибока | **Налаштування** | J2, E1 |

### Глобальна навігація (3 пункти)

```
Головна  ·  Пошук  ·  Дітям/Вийти
```

- **Головна** → J4 (відкриття нового) + Main (стартова точка без запиту)
- **Пошук** → J3 (фільтри inline) + Main (конкретний пошук за назвою)
- **Дітям/Вийти** → J2 (безпечний простір) + E1 (контроль). В дитячому режимі стає «Вийти» з PIN-підтвердженням.

### Main Flow — від запуску до перегляду

```
[Головна] → (browse: тап на постер) → [Картка фільму] → OTT
          → (search: Пошук + фільтри) → [Картка фільму] → OTT
```

### Глибина тапів до Main Job

| Сценарій | Тапи |
|----------|------|
| Режим активний → тап на мультфільм | **1** |
| Вмикає дитячий режим (PIN є) → тап | **2** |
| Пошук + фільтри → тап | **3** (межа) |
| Перший запуск (onboarding) | виняток |

### Відкриті рішення до wireframing

1. **H2 / OTT-посилання**: чи входить у MVP? Якщо так — секція «Де дивитись» у Картці фільму.
2. **Фільтри у Пошуку**: inline (завжди відкриті) чи slideup? Від цього залежить реальна глибина J3.
3. **PIN без акаунту**: дозволити анонімний PIN (local storage) або обов'язкова реєстрація?
